# Punch-RPG 클라이언트 UI 개발 및 색상 표준 지침 (UI Standards)

본 문서는 Punch-RPG의 클라이언트 UI 일관성을 보장하고, 의도치 않은 색상(예: 기본 노란색 호버 등)이 인터페이스에 침투하는 현상을 방지하기 위해 제정된 디자인 규격 및 개발 가이드라인입니다.

---

## 1. UI 핵심 색상 테마 및 변수 규격

프로젝트 UI 개발 시에는 글로벌 `UITheme.lua`를 기준으로 하며, 모듈별 로컬 변수를 오버라이드할 때도 반드시 아래 남색 계열 테마 색상 상수를 선언하여 통일성을 유지해야 합니다.

- **기본 패널 배경색 (`C.BG_PANEL`)**: `Color3.fromRGB(10, 15, 25)` (Deep Navy)
- **어두운 영역 배경색 (`C.BG_DARK`)**: `Color3.fromRGB(5, 5, 10)` (Pure Black)
- **슬롯 기본 배경색 (`C.BG_SLOT`)**: `Color3.fromRGB(12, 12, 15)` (Near Black)
- **기본 테두리 강조색 (`C.BORDER`)**: `Color3.fromRGB(60, 85, 130)` (Light Navy)
- **액션 버튼 배경색 (`C.BTN`)**: `Color3.fromRGB(40, 80, 160)` (Accent Navy)
- **액션 버튼 호버색 (`C.BTN_H`)**: `Color3.fromRGB(60, 100, 180)` (Bright Navy)
- **기본 활성/선택 테두리 (`C.GOLD_SEL`)**: `Color3.fromRGB(40, 80, 160)` (Accent Blue)
- **텍스트 색상**: 정보 및 문구 텍스트는 원칙적으로 하얀색(`C.WHITE`) 또는 회색(`C.GRAY`)을 사용해야 합니다. 
  * *주의*: 네이비 계열(`C.GOLD_SEL`)을 일반 텍스트 색상 매핑으로 사용하지 마십시오.

## 2. 버튼 호버 피드백 (`UIUtils.mkBtn`) 적용 표준

`UIUtils.mkBtn` 함수는 `hbg` (호버 배경색)가 명시되지 않거나 `isNegative` 매개변수가 없을 때, 글로벌 테마의 `Theme.Colors.BTN_H` (노란색, `255, 225, 90`)를 기본값으로 사용합니다. 따라서 다음 표준에 따라 속성을 반드시 정의해야 합니다.

### A. 액션/확인 버튼 (남색 계열)
- 마우스 오버 시 남색에서 은은하게 밝아지는 남색으로 피드백을 주어야 합니다.
- **필수 지정 속성**:
  ```lua
  bg = C.BTN,
  hbg = C.BTN_H, -- 절대 생략 금지 (생략 시 노란색 호버가 침투함)
  ```

### B. 시스템 제어 버튼 (닫기 X, 최소화, 접기/펼치기 및 취소 버튼)
- 마우스 오버 시 절대 노란색 피드백을 주어서는 안 되며, 은은한 회색 계열로 반응해야 합니다.
- **필수 지정 속성**:
  ```lua
  bg = C.BG_SLOT, -- 또는 C.BTN_GRAY
  hbg = C.BTN_GRAY_H,
  isNegative = true, -- true 지정 시 노란색 폴백 차단
  ```

### C. 배경이 투명한 텍스트형 제어 버튼 (`bgT = 1`)
- 튜토리얼 접기/펼치기 버튼처럼 배경 없이 텍스트 자체를 버튼으로 사용하는 경우, 호버 시 뒤쪽에 사각형 배경이 나타나면 안 됩니다.
- **필수 지정 속성**:
  ```lua
  bgT = 1,
  noHover = true, -- 호버 시 백그라운드 트윈 작동 완전 차단
  ```

## 3. 팝업 / 다이얼로그 모달 디자인 규격
- **다이얼로그 확인 버튼**: 노란색 배경(`C.BTN`)을 절대 사용하지 마십시오. 확인 버튼도 시스템 일관성을 맞추기 위해 남색 액션 버튼 규격(`bg = Color3.fromRGB(40, 80, 160) / hbg = Color3.fromRGB(60, 100, 180)`)으로 처리해야 합니다.
- **다이얼로그 취소 버튼**: 취소 버튼 역시 회색 호버 및 `isNegative = true` 속성을 명확히 세팅해야 합니다.
- **제목 텍스트**: 다이얼로그 타이틀 글자색은 항상 흰색(`C.WHITE`)으로 지정하여 가독성을 높입니다.

## 4. UI 검증 가이드
새로운 UI 요소 추가 또는 기존 UI 수정 시 반드시 아래 사항을 체크 리스트로 활용해 검증해 주십시오.
1. [ ] 마우스를 올렸을 때 노란색(`255, 225, 90`) 피드백이 나타나는 버튼이 단 하나라도 있는가?
2. [ ] X 버튼, 접기 버튼, 취소 버튼에 `isNegative = true`와 `hbg = C.BTN_GRAY_H`가 누락되었는가?
3. [ ] 텍스트만 보여야 하는 투명 버튼에 `noHover = true`가 적용되었는가?
4. [ ] 타이틀이나 일반 정보 텍스트가 하얀색이 아닌 네이비 블루 등 가독성이 떨어지는 색상으로 매핑되었는가?
